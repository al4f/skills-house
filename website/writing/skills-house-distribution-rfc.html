RFC: skills-house Distribution Model

Jul 2026 · Status: Draft for feedback

Author: al4f
Repo: skills-house
Goal: Define how skills-house distributes built Agent Skills via npm and CLI

Summary

Today, skills-house is clone-and-build. This RFC proposes npx skills add <name> to install individual skills from npm without cloning the monorepo.

Non-goals (v1 distribution)

  • Skills marketplace UI
  • Private registry support
  • Version pinning across skill dependency graphs

Package model

Each published skill is an npm package:

@skills-house/skill-skill-auditor
@skills-house/skill-my-skill

Package contents = built skills-dist/<name>/ layout only. No source, no build tooling.

CLI: npx skills add

npx skills add skill-auditor
npx skills add my-skill --agent cursor --scope project

Behavior:

  1. Download @skills-house/skill-<name> from npm
  2. Extract to temp directory
  3. Install to agent paths (reuse install-skills.sh logic)
  4. Print success + link to skill docs

Metrics philosophy

Per-skill npm packages enable weekly download counts — useful signal, not a success metric. Reputation comes from educational content and DX, not download leaderboards.

Sequencing

Ship distribution only after:

  • Flagship article + al4f.dev authority hub live
  • CI on PRs
  • At least two reference skills in catalog

Feedback

Open a GitHub issue with tag distribution-rfc or reply on the announcement thread.
